# Scenic Route Planner

Plan **scenic driving routes worldwide** by scoring the landscape from satellite
imagery, terrain and land cover, then routing A→B along the most scenic real‑road
path — not just the fastest one. The UK remains the densest catalogue of named
regions and presets; scoring and explore attractors work globally.

Everything runs on **free, no‑API‑key** services (Esri World Imagery, OSRM,
Open‑Meteo / Opentopodata elevation, OpenStreetMap Overpass land cover).

## Status

**Local MVP — worldwide product surface, not yet “fine anywhere”.** Colour
scoring is climate-mapped worldwide (`app/climates.py`: 12 full HSV climates +
offline 1° grid; UK temperate oceanic numbers locked). Arid canyon reds,
Mediterranean scrub, savannah gold, alpine snow/rock and polar ice score as
scenic without requiring lush green; urban grey stays low in the same climates.
Map context uses a fixed global OSM tag pack with climate-aware multipliers;
terrain softening lifts flat but scenic desert / alpine / polar contexts.
Field mode reserves a colour wall-clock slice and adapts OSRM reserve on warm
corridors so cold Overpass is less likely to leave heatmaps proxy-only forever.
Explore mode diverts through pluggable scenic attractors (UK parks + world packs).
Region jump and featured presets include non-UK demos. DIY bind is `127.0.0.1`;
see [`docs/THREAT_MODEL.md`](docs/THREAT_MODEL.md) and
[`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md).

**“Fine anywhere” status:** semantics and calibration are worldwide-ready;
first-plan quality on cold public Overpass/OSRM corridors remains the main gap.
Honest remaining SPOFs: public OSRM + Overpass rate limits. Friends-demo GO for
known warm corridors; **No-Go** for claiming every landscape type works equally
on a cold first hit under the default ~30s budget.

**Share with friends:** public OSRM by default (no Docker). Set a host key, bind
on LAN/VPS with **`--workers 1`**, prewarm featured land cover, run
`smoke-friends`, then invite — full checklist in
[`docs/FRIENDS_DEMO.md`](docs/FRIENDS_DEMO.md). Optional BYO router URL is
documented there; regional extract / compose OSRM is advanced appendix only
([`docs/SELF_HOSTED_OSRM.md`](docs/SELF_HOSTED_OSRM.md)).

## Highlights

- **Real‑road routing** via OSRM with scenic alternatives injected through scenic
  "hotspots" — not an abstract grid.
- **Three‑signal scenic score** blended per route:
  - **Colour** — satellite tile downscaled and analysed in HSV (green/blue =
    scenic, grey/low‑saturation = built‑up).
  - **Terrain** — elevation *relief* (hilly/mountainous drives score higher).
  - **Land cover** — proximity to forest, water, coast, peaks, parks and nature
    reserves boosts the score; industrial / residential / retail lowers it.
- **Live search stream** — watch candidate routes draw onto the map in real time
  as the planner explores them (Server‑Sent Events), including land‑cover
  tile‑loading progress.
- **Minimum‑scenic target** — set a hard floor (0–100); the planner *escalates*
  its search (wider corridor, longer detours, chained hotspots) until a route
  meets it.
- **Explore‑everything mode** — disregards travel time entirely and deliberately
  diverts through nearby **scenic attractors** (national parks and world hotspots),
  ranking routes purely by scenery.
- **Harsh motorway avoidance** — a heavy per‑km penalty pushes routes off
  motorways.
- **Per‑road explainability** — hover a road segment to see *why* it's scenic
  (colour vs terrain vs land cover, dominant nearby feature, elevation).
- **Roads‑style base map** (Google‑Maps‑like) for easy navigation.

## How it works

```
        satellite tile            elevation            OSM land cover
        (Esri, no key)          (Open‑Meteo/           (Overpass, no key,
              │                  Opentopodata)          cached to disk)
              ▼                       │                       │
     downscale + HSV colour      relief score            proximity score
              └───────────────┬──────┴───────────────────────┘
                              ▼
                   per‑sample scenic blend (0–100)
                              ▼
   OSRM real‑road candidates  ──►  score every route  ──►  rank + pick
   (base + hotspot detours +        (scenic / cost /         the best route
    attractor diversions)            hard target)            for your settings
```

1. **Candidate routes** (`app/roads.py`) — OSRM returns the base route plus
   alternatives; extra candidates are generated by routing through scenic
   hotspots and (in explore mode) through scenic attractors.
2. **Scoring** (`app/roads.py` → `app/scoring.py`, `app/enrich.py`) — each route
   is sampled along its length and scored on colour + terrain + land cover,
   distance‑weighted into a single 0–100 scenic score with a per‑segment
   explanation.
3. **Ranking** — normal mode blends travel time with scenic quality
   (`cost = duration × (1 + preference × detour × (1 − scenic/100)) + motorway_penalty`);
   explore mode ranks by **scenic score only**.
4. **API + UI** (`app/main.py`, `frontend/`) — FastAPI backend + Leaflet map.

### Land‑cover tile cache (important)

A single Overpass query over a long corridor times out, so land cover is fetched
on a **fixed global 0.6° tile grid** and **cached to disk**
(`data/landcover_cache/`). Each tile is fetched from Overpass **at most once
ever**, then served instantly. The **first** route through a brand‑new area is
therefore slower while cold tiles download (the UI shows tile‑loading progress);
every subsequent route in that area is fast. Elevation results are cached in-process and on disk (`data/elev_cache/`) and
fall back to a second provider when one hits its daily quota.

## Quick start

```powershell
pip install -r requirements.lock.txt
# Looser pins: pip install -r requirements.txt

# Or use the helper:
#   .\scripts\dev.ps1 run

python -m uvicorn app.main:app --host 127.0.0.1 --port 8077

# Open http://127.0.0.1:8077
#   • search or click to set Start (A) and End (B)
#   • pick a scenery style, optional min‑scenic target, avoid‑motorways,
#     and explore‑everything
#   • watch the live search, then switch between the returned routes
```

```powershell
python -m pytest -q
# or: .\scripts\dev.ps1 test
```

[`docs/API_SURFACE.md`](docs/API_SURFACE.md).

python scripts/biome_calibration.py
python scripts/biome_calibration.py --json

```powershell
.\scripts\dev.ps1 run
# in another shell:
python scripts/demo_corridor_qa.py --base http://127.0.0.1:8077
python scripts/demo_corridor_qa.py --quick   # 12 corridors for faster dogfood

## API

Full supported vs experimental table: [`docs/API_SURFACE.md`](docs/API_SURFACE.md).

| Endpoint | Purpose |
|----------|---------|
| `GET /api/health` | status, cells, `osrm_configured`, `cache_entries` |
| `GET /api/route` | scenic route on real roads (JSON) |
| `GET /api/route/stream` | same planner, streamed as live Server‑Sent Events |
| `GET /api/route/compare` | fastest vs most‑scenic |
| `GET /api/geocode?q=` | place search (Nominatim proxy) |
| Saves / history / export / presets / regions / profiles | see API_SURFACE |

**Experimental (not wired in the UI):** `/api/poi`, `/api/features`, `/api/cells`
(+ `scripts/build_grid.py` / `app/graph.py` lattice). The UI uses curated scenery
styles and `FEATURED_PRESET_IDS` only — full profile/preset catalogues remain on
the API for advanced use.

**Route query parameters** (`/api/route` and `/api/route/stream`):

| Param | Default | Meaning |
|-------|---------|---------|
| `from_lat`, `from_lng`, `to_lat`, `to_lng` | – | start (A) and end (B) |
| `preference` | `0.7` | 0 = fastest → 1 = most scenic |
| `profile` | `balanced` | scenery profile (weights + detour appetite) |
| `avoid_motorways` | `false` | hard ban on motorway km when any non-motorway option exists |
| `min_scenic` | `0` | hard floor (0–100); unmet → most scenic (not fastest) |
| `explore_all` | `false` | disregard time; divert through national parks, rank by scenery |
| `time_budget` | `true` | when false, skip the ~30s wall-clock early-exit |

Compare (`/api/route/compare` and `/api/route/compare/stream`) accepts the same
knobs for the scenic leg; the fastest leg always uses `preference=0`.

The result contains a `chosen` route plus `alternatives`, each with
`avg_scenic_score`, a `components` breakdown (colour / terrain / landcover),
`distance_km`, `duration_min`, `motorway_km`, turn‑by‑turn steps and a
`render` polyline carrying per‑segment scenic scores and explanations.
Soft degradation: `budget_exhausted`, `budget_reasons`, `min_scenic_met`,
`motorway_avoid_met`, `time_budget`, `signals.landcover_incomplete`.

## Configuration

All tunables are in `app/config.py`, including colour thresholds, signal blend
weights, motorway penalty, corridor padding, the hard‑target escalation rounds,
the explore‑mode budgets, the attractor registry, and the land‑cover / elev
cache directories. See [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md) for details.

**Upstream env overrides** (defaults = public free services):

| Variable | Purpose |
|----------|---------|
| `SCENIC_OSRM_URL` | Optional BYO OSRM URL template with `{coords}` (unset = public) |
| `SCENIC_OVERPASS_URLS` | Comma-separated Overpass interpreter URLs |
| `SCENIC_NOMINATIM_URL` | Nominatim search endpoint |
| `SCENIC_LOG_LEVEL` | Python log level (default `INFO`) |
| `SCENIC_TILE_ZOOM` / `SCENIC_DOWNSCALE` | Colour tile analysis |
| `SCENIC_MOTORWAY_PENALTY` | Virtual minutes per motorway km |
| `SCENIC_CACHE_MAX_ENTRIES` / `SCENIC_CACHE_TTL_SEC` | In-process elevation/landcover memo |
| `SCENIC_PUBLIC` / `SCENIC_BIND_HOST` | Enable public gate (non-loopback) |
| `SCENIC_API_KEY` | Required when public gate is on (`X-API-Key` / `api_key`) |
| `SCENIC_RATE_LIMIT_PER_MIN` | Per-IP rate limit in public mode (default 30) |
| `SCENIC_PLAN_BUDGET_SEC` | Soft plan wall-clock budget (default 30) |
| `SCENIC_MAX_INFLIGHT_PLANS` | Concurrent plan/stream cap (default 2) |

Friends checklist: [`docs/FRIENDS_DEMO.md`](docs/FRIENDS_DEMO.md). Optional BYO

## Notes & limitations

- **Localhost open by default.** Bind to `127.0.0.1` for DIY use (no auth).
  Non-loopback / `SCENIC_PUBLIC=1` requires `SCENIC_API_KEY` — see
  [`docs/THREAT_MODEL.md`](docs/THREAT_MODEL.md) and
  [`docs/FRIENDS_DEMO.md`](docs/FRIENDS_DEMO.md).
- **Free public services** (Overpass, OSRM demo, Open‑Meteo) are rate‑limited and
  occasionally slow; the disk caches and provider fallbacks make the app tolerant
  of this, but the very first route in a new area can be slow while tiles warm.
- **Scenic field (beta)** paints a wide heatmap before OSRM (default cell ~0.006°,
  finer than the old 0.015° grid). Urban grey/white and town-park greens are
  hard-rejected from corridor spines (`FIELD_REJECT_SCENIC`); discarded corridors
  show up in `field_meta.corridors_rejected`. On cold corridors the heatmap may
  still stay partly **proxy** (terrain + map context) under the ~30s budget; a
  guaranteed colour floor + adaptive OSRM reserve reduce true proxy-only
  outcomes. The UI and `field_meta.heatmap_proxy_only` (zero colour) say so
  explicitly when colour never landed.
- **Avoid motorways** is best-effort on long inter-city corridors (e.g. London–Bath,
  Denver–Aspen): when no motorway-free candidate exists within budget, the UI shows
  `motorway_avoid_reason: all_candidates_include_motorways` and the least-motorway
  option is returned honestly.
- **Explore attractors** use UK parks plus compact world packs
  (`app/attractors.py`); corridors without a nearby seed still score via land
  cover + relief hotspots.
- This is a self‑contained MVP; friends demo first, then optional self-host —
  see [`docs/PRODUCTION.md`](docs/PRODUCTION.md).
