# 🗺️ Scenic Route Planner

Plan **scenic driving routes** across the UK (and, in principle, anywhere) by
scoring the landscape from satellite imagery, terrain and land cover, then
routing A→B along the most scenic real‑road path — not just the fastest one.

Everything runs on **free, no‑API‑key** services (Esri World Imagery, OSRM,
Open‑Meteo / Opentopodata elevation, OpenStreetMap Overpass land cover).

---

## Highlights

- **Real‑road routing** via OSRM with scenic alternatives injected through scenic
  "hotspots" — not an abstract grid.
- **Three‑signal scenic score** blended per route:
  - **Colour** — satellite tile downscaled and analysed in HSV (green/blue =
    scenic, grey/low‑saturation = built‑up).
  - **Terrain** — elevation *relief* (hilly/mountainous drives score higher).
  - **Land cover** — proximity to forest, water, coast, peaks, parks and nature
    reserves boosts the score; industrial / residential / retail lowers it.
- **Live search stream** — watch candidate routes draw onto the map in real time
  as the planner explores them (Server‑Sent Events), including land‑cover
  tile‑loading progress.
- **Minimum‑scenic target** — set a hard floor (0–100); the planner *escalates*
  its search (wider corridor, longer detours, chained hotspots) until a route
  meets it.
- **Explore‑everything mode** — disregards travel time entirely and deliberately
  diverts through nearby **UK National Parks**, ranking routes purely by scenery.
- **Harsh motorway avoidance** — a heavy per‑km penalty pushes routes off
  motorways.
- **Per‑road explainability** — hover a road segment to see *why* it's scenic
  (colour vs terrain vs land cover, dominant nearby feature, elevation).
- **Roads‑style base map** (Google‑Maps‑like) for easy navigation, with an
  optional scenic heatmap overlay.

## How it works

```
        satellite tile            elevation            OSM land cover
        (Esri, no key)          (Open‑Meteo/           (Overpass, no key,
              │                  Opentopodata)          cached to disk)
              ▼                       │                       │
     downscale + HSV colour      relief score            proximity score
              └───────────────┬──────┴───────────────────────┘
                              ▼
                   per‑sample scenic blend (0–100)
                              ▼
   OSRM real‑road candidates  ──►  score every route  ──►  rank + pick
   (base + hotspot detours +        (scenic / cost /         the best route
    national‑park diversions)        hard target)            for your settings
```

1. **Candidate routes** (`app/roads.py`) — OSRM returns the base route plus
   alternatives; extra candidates are generated by routing through scenic
   hotspots and (in explore mode) through national parks.
2. **Scoring** (`app/roads.py` → `app/scoring.py`, `app/enrich.py`) — each route
   is sampled along its length and scored on colour + terrain + land cover,
   distance‑weighted into a single 0–100 scenic score with a per‑segment
   explanation.
3. **Ranking** — normal mode blends travel time with scenic quality
   (`cost = duration × (1 + preference × detour × (1 − scenic/100)) + motorway_penalty`);
   explore mode ranks by **scenic score only**.
4. **API + UI** (`app/main.py`, `frontend/`) — FastAPI backend + Leaflet map.

### Land‑cover tile cache (important)

A single Overpass query over a long corridor times out, so land cover is fetched
on a **fixed global 0.6° tile grid** and **cached to disk**
(`data/landcover_cache/`). Each tile is fetched from Overpass **at most once
ever**, then served instantly. The **first** route through a brand‑new area is
therefore slower while cold tiles download (the UI shows tile‑loading progress);
every subsequent route in that area is fast. Elevation results are cached and
fall back to a second provider when one hits its daily quota.

## Quick start

```powershell
pip install -r requirements.txt

# Run the app
python -m uvicorn app.main:app --host 127.0.0.1 --port 8077

# Open http://127.0.0.1:8077
#   • search or click to set Start (A) and End (B)
#   • pick a scenery profile, optional min‑scenic target, avoid‑motorways,
#     and explore‑everything
#   • watch the live search, then switch between the returned routes
```

> Real‑road routing and enrichment work out of the box — no grid build required.
> Optionally, precompute a scored **heatmap grid** for a region:
>
> ```powershell
> python -m scripts.build_grid --preset lake-district
> # or a custom box:
> python -m scripts.build_grid --min-lat 54.2 --min-lng -3.4 --max-lat 54.7 --max-lng -2.7 --cell 0.02
> ```
> Presets: `lake-district`, `snowdonia`, `peak-district`, `london`,
> `scotland-highlands`. Run fully offline with `--source synthetic`.

## API

| Endpoint | Purpose |
|----------|---------|
| `GET /api/health` | status + scored cell count |
| `GET /api/meta` | grid metadata |
| `GET /api/cells?min_lat=&min_lng=&max_lat=&max_lng=` | scored cells for the heatmap (bbox optional) |
| `GET /api/score?lat=&lng=` | live scenic score for any coordinate |
| `GET /api/route` | scenic route on real roads (JSON) |
| `GET /api/route/stream` | same planner, streamed as live Server‑Sent Events |

**Route query parameters** (`/api/route` and `/api/route/stream`):

| Param | Default | Meaning |
|-------|---------|---------|
| `from_lat`, `from_lng`, `to_lat`, `to_lng` | – | start (A) and end (B) |
| `preference` | `0.7` | 0 = fastest → 1 = most scenic |
| `profile` | `balanced` | scenery profile (weights + detour appetite) |
| `avoid_motorways` | `false` | apply a harsh per‑km motorway penalty |
| `min_scenic` | `0` | hard floor (0–100); search escalates until met |
| `explore_all` | `false` | disregard time; divert through national parks, rank by scenery |

The result contains a `chosen` route plus `alternatives`, each with
`avg_scenic_score`, a `components` breakdown (colour / terrain / landcover),
`distance_km`, `duration_min`, `motorway_km`, turn‑by‑turn steps and a
`render` polyline carrying per‑segment scenic scores and explanations.

Additional feature routers live in `app/saved.py`, `app/catalog_data.py`,
`app/exports.py` and `app/features.py` (saved routes, scenery profiles, exports).

## Configuration

All tunables are in `app/config.py`, including colour thresholds, signal blend
weights, motorway penalty, corridor padding, the hard‑target escalation rounds,
the explore‑mode budgets, the national‑park list, and the land‑cover tile size /
cache directory. See [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md) for details.

## Notes & limitations

- **Free public services** (Overpass, OSRM demo, Open‑Meteo) are rate‑limited and
  occasionally slow; the disk caches and provider fallbacks make the app tolerant
  of this, but the very first route in a new area can be slow while tiles warm.
- **National parks are UK‑only.** Elsewhere the planner still finds scenery via
  land cover + relief hotspots.
- This is a self‑contained MVP; for production consider an OSM road graph you
  control (Valhalla/GraphHopper), Sentinel‑2 rasters, and PostGIS/Redis.
